⌨️ Copilot CLI vs Claude Code

A developer's reference for slash commands, CLI flags, and keyboard shortcuts

Concepts & Modes

High-level concepts that work differently across the two tools. Understand these before diving into individual commands.

Permission & Interaction Modes

Copilot CLI

Cycle through modes with Shift+Tab:

Normal Default. The agent asks for permission before running commands that modify or execute files.
Plan The agent only proposes plans and explains what it would do — no file changes until you approve.
Autopilot The agent works autonomously until the task is complete. No permission prompts. (Experimental — enable with --experimental)

Launch flags: copilot --allow-all or copilot --yolo to skip all permission prompts for a session.

Claude Code

Cycle through modes with Shift+Tab:

Default Asks before running tools that modify files or execute commands.
Accept Edits Auto-approves file edits but still asks for shell commands.
Plan Read-only analysis and planning — no modifications until you switch modes.
Auto Classifier-based: safe commands run automatically, risky ones still prompt. Configurable via auto-mode rules.
Don't Ask Runs everything without prompting. Same as --dangerously-skip-permissions.
Bypass Permissions Full bypass including hooks. Requires --allow-dangerously-skip-permissions flag to appear in the cycle.

Launch flag: claude --permission-mode plan|auto|dontAsk to start in a specific mode.

Effort & Quality Control

Copilot CLI

Model Selection Choose between models (Claude Sonnet, GPT-5) via /model. Each model has different reasoning depth — that is how you control quality.
No explicit effort dial Quality is controlled by model choice. No low/medium/high effort toggle.

Claude Code

Effort Levels Fine-grained control: low → medium → high → xhigh → max. Adjustable mid-session via /effort or arrow keys in the model picker.
Fast Mode /fast on optimizes for speed — shorter, more concise responses. Good for rapid iteration.
Budget Caps --max-budget-usd 5 and --max-turns 10 limit spending and iterations in scripted/CI usage.

Session Lifecycle

Copilot CLI

Foreground only Sessions run in your terminal. You can delegate to the cloud via /delegate for async PR creation, but local sessions are foreground-only.
Delegate to Cloud /delegate sends the session to GitHub's cloud agent, which continues asynchronously and creates a PR.
Fleet Mode /fleet spawns parallel sub-agents within the session for concurrent task execution.

Claude Code

Foreground + Background Sessions can run in foreground or be detached to background with /bg. Multiple background agents run in parallel.
Background Agents claude --bg "task" or /background detaches to background. Monitor with claude agents, reattach with claude attach <id>.
Batch / Worktrees /batch decomposes work into 5–30 units, spawns one agent per unit in isolated git worktrees, each opening its own PR.
Goal-driven /goal "all tests pass" makes the agent work autonomously across multiple turns until the success condition is met.

Custom Instructions & Memory

Copilot CLI

Instruction Files Reads CLAUDE.md, GEMINI.md, AGENTS.md (git root & cwd), .github/copilot-instructions.md, and .github/instructions/**/*.instructions.md
User-level ~/.copilot/copilot-instructions.md for global instructions across all projects.
No auto-memory Instructions are manually authored. No automatic learning from sessions.

Claude Code

CLAUDE.md Primary instruction file at project root, home directory (~/.claude/CLAUDE.md), and nested directories.
Auto-memory Claude can automatically learn from sessions and write entries to memory files. Toggle with /memory.
Settings Files .claude/settings.json for project config, ~/.claude.json for user-level config. Controls permissions, tools, and behavior.

CI & Scripting

Copilot CLI

Interactive only Designed for interactive terminal sessions. No built-in non-interactive/print mode for scripting.
Cloud delegation For CI-like workflows, use /delegate to send work to the cloud Copilot coding agent.

Claude Code

Print Mode (-p) Non-interactive mode: claude -p "query". Supports piping (cat file | claude -p), JSON output, budget caps, and turn limits.
Structured Output --output-format json and --json-schema for machine-parseable responses in CI pipelines.
OAuth Tokens claude setup-token generates long-lived tokens for CI. --bare mode skips discovery for faster startup.

TL;DR — Key Differences

Copilot CLI strengths

  • Deep GitHub integration — # mentions for issues/PRs
  • /delegate to hand off to cloud agent for PR creation
  • /fleet mode for parallel sub-agents
  • Built-in LSP server support via /lsp
  • /research for deep web + GitHub search
  • Autopilot mode (/autopilot)
  • /share to gist, HTML, or markdown
  • Reads CLAUDE.md, GEMINI.md, and AGENTS.md

Claude Code strengths

  • Print mode (-p) for scripting & CI pipelines
  • /batch for large-scale parallel refactors via worktrees
  • /ultrareview & /ultraplan — cloud-powered deep analysis
  • /goal — autonomous until condition is met
  • Effort tuning (low → max)
  • Budget caps (--max-budget-usd)
  • Voice dictation, Chrome integration, desktop app
  • Background agents with claude --bg
  • /autofix-pr — auto-push fixes on CI failure
